QueueMe Accessibility Test Checklist

Use this checklist to manually verify accessibility compliance. Test with:

  • Keyboard only (no mouse)
  • Screen reader (NVDA on Windows, VoiceOver on Mac)
  • Axe DevTools browser extension

1. Keyboard Navigation

Test Expected Result Pass Notes
Tab through all interactive elements All buttons, inputs, links receive focus in logical order
Skip link works Pressing Tab once shows "Skip to main content" link; activating it skips header
No keyboard traps Can always Tab/Shift+Tab out of any element
Modal focus trap When modal open, Tab cycles within modal only; Escape closes modal
Grid arrow key navigation Arrow keys move between grid cells
Grid Home/End keys Home goes to first cell in row, End to last; Ctrl+Home/End for first/last cell in grid
Enter to edit grid cell Pressing Enter on cell opens edit mode
Escape to cancel edit Pressing Escape in edit mode cancels and restores original value
Global shortcuts Ctrl+S exports, Ctrl+N new session, Ctrl+I import

2. Focus Visibility

Test Expected Result Pass Notes
Buttons have visible focus All buttons show clear focus ring (at least 2px)
Inputs have visible focus All text inputs show clear focus ring
Grid cells have visible focus Focused grid cell is clearly highlighted
Error links have visible focus Error summary links show focus when tabbed to

3. Form Accessibility

Test Expected Result Pass Notes
All inputs have labels Every input has an associated label (visible or visually-hidden)
Required fields marked Required fields have visual indicator and aria-required="true"
Error fields marked invalid Fields with errors have aria-invalid="true" and visible error styling
Error messages associated Error messages connected via aria-describedby

4. Error Handling

Test Expected Result Pass Notes
Inline errors appear Submitting invalid form shows error messages next to fields
Error summary appears Top-of-page error summary lists all errors
Error summary has role="alert" Summary has role="alert" for screen reader announcement
Error links work Clicking error in summary focuses the problematic field
Screen reader announces errors NVDA/VoiceOver announces "X errors must be corrected"

5. Screen Reader Testing

Test Expected Result Pass Notes
Page title announced Screen reader announces "QueueMe - Live365 CSV Generator"
Landmarks present Screen reader can navigate by landmarks (header, main, footer)
Headings hierarchy H1 > H2 structure is correct and navigable
Button labels announced All buttons announce their purpose
Grid announces cell content Moving through grid announces column header and cell value
Edit mode announced Entering/exiting edit mode is announced
Row added announced After adding row, success message is announced
Export success announced After export, success message is announced

6. Color and Contrast

Test Expected Result Pass Notes
Text contrast ratio All text meets 4.5:1 contrast ratio (use axe DevTools)
Error colors have text labels Errors are not indicated by color alone - also have text/icon
Focus visible without color Focus indicators visible in grayscale mode

7. Automated Testing (Axe DevTools)

Test Expected Result Pass Notes
No critical issues Axe reports 0 critical accessibility issues
No serious issues Axe reports 0 serious accessibility issues
Review moderate issues Any moderate issues reviewed and addressed or documented
